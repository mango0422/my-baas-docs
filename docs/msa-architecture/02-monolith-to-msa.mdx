| 모놀리식 한계               | MSA 해결책                  |
| --------------------------- | --------------------------- |
| **전체 수직 확장** 필요     | 필요한 서비스만 수평 확장   |
| 작은 수정도 **전체 재배포** | **독립 배포** 로 다운타임 ↓ |
| **기술 종속**               | 서비스별 **Polyglot**       |
| 코드 충돌·빌드 지연         | **작은 팀** 단위 병렬 개발  |
| 장애 전파 위험              | **서킷 브레이커** 로 격리   |

### 전환 배경 한눈에 보기

1. **개발 생산성** : 작은 코드베이스 → 빠른 코드 리뷰·테스트
2. **확장 비용** : 트래픽 집중 서비스만 스케일-아웃
3. **배포 안정성** : 실패 시 영향 범위 최소
4. **기술 혁신** : 서비스별 최적 기술 스택 도입
