### 3.1 데이터 중복은 **필연적**

- 회원·주문·배송 서비스가 **고객 정보**를 각각 보유
- 비즈니스 성능을 위해 로컬 읽기 모델 필요 → _최종 일관성_ 수용

### 3.2 Authority DB(원천 DB)가 필요한 4가지 이유

1. **정합성 보장** : 모든 쓰기는 원천 DB 경유 → ACID 확보
2. **Idempotency** : 재처리 여부 판단을 로컬 트랜잭션 로그로 해결
3. **컴플라이언스** : 변경 이력·감사 추적을 서비스 내부에 저장
4. **SLA** : 잔액 조회 \< 10ms 등 초저지연 요구 → 로컬 DB 필수

### 3.3 동기화 전략

- **이벤트** 기반(CDC, Kafka) → 실시간 반영
- **API 콜** → 최신 데이터 즉시 조회
- **배치** → 통계·보고용 대량 동기화
