> 금융 시스템은 **보안·정합성·규제 준수** 가 생명입니다.

## 4.1 보안(Security)

### 4.1.1 인증·인가

- **강화된 사용자 인증** : MFA·생체인증, 비정상 로그인 탐지
- **서비스 간 mTLS + JWT/OAuth2**
- **세분화 RBAC** : 최소 권한 원칙

### 4.1.2 암호화

| 구분        | 적용                                    |
| ----------- | --------------------------------------- |
| **전송 중** | TLS 1.3, 강력 CipherSuite               |
| **저장 시** | DB TDE / AES-256, 파일·백업 전면 암호화 |
| **키 관리** | KMS·HSM, 주기적 Key Rotation            |

### 4.1.3 API 보안

- API Gateway로 인증·Rate Limiting·WAF
- 입력값 검증, OWASP Top 10 방어
- DDoS 탐지·차단

---

## 4.2 데이터 정합성

### 4.2.1 트랜잭션 관리

- **Saga 패턴** + 보상 트랜잭션
- 강한 일관성(계정계) ↔ 최종 일관성(서비스 간)

### 4.2.2 데이터 동기화

- 실시간 vs 배치 기준 정의
- **CDC (Debezium)** 활용
- 해시 비교·Reconciliation Job

### 4.2.3 데이터 거버넌스

- 데이터 소유자 명확화·카탈로그
- 데이터 품질 지표 & 정제 프로세스
- 엄격한 백업·DR 테스트

---

## 4.3 규제 준수·감사

1. **법규 준수** : 전자금융, 개인정보보호, AML 등
2. **감사 Trail** : 모든 거래·이벤트 로그 WORM 저장, 위변조 방지
3. **보고 자동화** : 감독기관 보고서·실시간 이상징후 탐지

---

## 4.4 성능·안정성

- **Circuit Breaker / Retry / Timeout**
- 비동기 메시징으로 부하 분산
- 무상태 서비스 + 오토스케일링(K8s)
- 분산 추적(Jaeger) + 중앙 로깅(ELK / Loki)

---

## 4.5 운영·관리

- **CI/CD 파이프라인** + GitOps
- **Blue-Green / Canary** 배포
- API Gateway  + 버전 관리·문서화
- **2-Pizza Team** : 서비스별 독립 책임
