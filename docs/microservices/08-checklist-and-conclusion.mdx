# 도입 전 셀프 체크리스트

1. **비즈니스 도메인**이 명확하게 분할돼 있는가?
2. 조직이 **CI/CD·관측성·보안 자동화**를 지속 운영할 수 있는가?
3. **운영 복잡도 ↔ 얻을 이점**이 균형적인가?
4. 데이터 **분산 일관성**을 감당할 기술·조직 역량이 있는가?
5. 서비스 수 ≤ 10 이하면 **모놀리스 + 모듈화**가 낫지 않은가?

## 결론 — “작게 시작해, 자동화로 확신하라”

MSA는 **코드 설계**만이 아니라 **조직·문화·프로세스**까지 포괄하는 총체적 접근입니다.  
작은 서비스에서 출발해 **자동화·관측성·보안**을 먼저 체계화하면, 서비스 수가 기하급수적으로 늘어나도 운영 혼란 없이 확장·진화할 수 있습니다.

> **핵심 메시지**  
> _“서비스는 자율적으로, 운영은 표준적으로.”_
