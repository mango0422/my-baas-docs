# 마이크로서비스 아키텍처(Microservices Architecture, MSA) 소개

> **한마디 정의**  
> _“비즈니스 기능별로 **잘게 쪼개고(De-compose)**, **느슨하게 연결(Loosely-Coupled)** 하여 **독립 개발·배포·확장** 가능한 서비스 집합”_

## 왜 MSA인가?

- **민첩성**: 서비스 단위 빠른 변경·릴리스
- **확장성**: 서비스별 수평/수직 확장 독립
- **복원력**: 장애 격리, 부분 실패 허용
- **기술 다양성**: 팀 혹은 도메인별 최적 스택 선택

## 단일 모놀리스 vs MSA

| 항목         | 모놀리스      | 마이크로서비스   |
| ------------ | ------------- | ---------------- |
| 배포 단위    | 1개 (전체)    | 기능별 개별      |
| 장애 영향    | 전체 영향     | 국소 영향        |
| 확장 방식    | 전체 Scale-up | 부분 Scale-out   |
| 코드베이스   | 거대, 얽힘    | 작고 명확한 경계 |
| 데이터베이스 | 단일 스키마   | DB-per-service   |

---

> **TIP**: 팀 규모·도메인 복잡도가 충분히 크지 않다면, 모놀리스 + 모듈화가 오히려 효율적입니다.
